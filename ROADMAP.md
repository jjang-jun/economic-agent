# Economic Agent Roadmap

## North Star
경제적 자유를 목표로, 이 프로젝트는 단순 뉴스 요약기가 아니라 아래 루프를 계속 개선하는 투자 의사결정 보조 시스템이 되어야 한다.

```
시장/공시/가격 데이터 수집
  -> 시장 레짐 판단
  -> 종목 후보와 리스크 도출
  -> 내 포트폴리오 기준 행동 가드레일 적용
  -> 추천/실행/성과 기록
  -> 실패와 성공 패턴 학습
```

## 원칙
- AI는 최종 매수/매도 버튼이 아니라 근거 정리, 누락 리스크 탐지, 시나리오 비교에 사용한다.
- 매수 추천보다 리스크 관리와 성과 검증을 우선한다.
- 모든 추천은 저장하고 1일/5일/20일 성과와 KOSPI 대비 초과수익률로 검증한다.
- 히스토리는 캐시가 아니라 장기 저장소에 쌓아야 한다.
- 시장이 나쁠 때는 무엇을 살지보다 얼마나 쉬어야 하는지를 먼저 판단한다.

## 현재 가장 중요한 다음 작업
1. 일일 행동 리포트를 만든다: 신규 매수, 보유 유지, 비중 축소, 매도 후보를 분리한다.
2. 성과 리뷰를 고도화한다: 섹터, 신호, 리스크팩터, 프롬프트 버전별 승률과 초과수익률을 비교한다.
3. 대시보드를 고도화한다: 기간 필터, 추천 상세, 실제 거래 상세, 리스크 차단 사유를 볼 수 있게 한다.
4. 브리핑/리포트 입력을 로컬 JSON fallback보다 Supabase 조회 중심으로 전환한다.
5. 실제 첫 거래부터 `trade:record`로 기록하고 `portfolio:sync-secret`으로 Actions 포트폴리오를 동기화한다.

## 운영 루프

실전 운용은 아래 순서를 기준으로 한다.

1. 추천 후보 확인: `npm run recommendations:list`
2. 실제 매매 기록: `npm run trade:record -- --side buy --symbol 005930 --name 삼성전자 --qty 1 --price 70000`
3. GitHub Actions 포트폴리오 동기화: `npm run portfolio:sync-secret`
4. 평가손익 스냅샷: `npm run portfolio:snapshot`
5. 실제 거래 성과 확인: `npm run trade:performance`
6. 주간/월간 복기: `npm run review:weekly`, `npm run review:monthly`
